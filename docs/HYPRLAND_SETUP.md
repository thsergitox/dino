# Hyprland setup

Starting in **v0.2**, `dino` is a persistent Textual TUI that lives in a Hyprland special workspace. You toggle the floating window with `SUPER+Z` and use Space inside the TUI to start/stop recording. The previous push-to-talk CLI flow is still available as "legacy" — covered at the end.

## The fast path — `dino setup`

```bash
dino setup
```

The wizard:

1. Asks language (Spanish default).
2. Asks lifecycle (`exec-once` recommended — dino is always ready; or `lazy-spawn` to start it only when you press the hotkey).
3. Auto-detects your terminal (kitty / alacritty / foot / ghostty / wezterm).
4. Appends the right Hyprland block to `~/.config/hypr/hyprland.conf`.
5. (If lazy lifecycle) writes `~/.config/dino/dino-toggle.sh`.

Then:

```bash
hyprctl reload
```

Press `SUPER+Z`. The dino terminal appears centered, floating. Press Space → speak → Press Space → transcript appears in the history pane and on your clipboard. `Ctrl+V` anywhere.

## Manual binding (exec-once)

If you'd rather paste it yourself:

```ini
exec-once = kitty --class dino --override hide_window_decorations=yes dino tui

windowrule = workspace special:dino silent, class:^(dino)$
windowrule = float, class:^(dino)$
windowrule = size 60% 50%, class:^(dino)$
windowrule = center 1, class:^(dino)$

bind = SUPER, Z, togglespecialworkspace, dino
```

Replace `kitty …` with your terminal — see the table below.

## Per-terminal launch commands

| Terminal | Command | Hide-decorations strategy |
|---|---|---|
| kitty | `kitty --class dino --override hide_window_decorations=yes dino tui` | flag above |
| alacritty | `alacritty --class dino,dino -e dino tui` | set `window.decorations = "None"` in `alacritty.toml` |
| foot | `foot --app-id=dino dino tui` | mostly decoration-free under Hyprland already |
| ghostty | `ghostty --class=dino --window-decoration=false -e dino tui` | flag above |
| wezterm | `wezterm start --class dino -- dino tui` | set `window_decorations = "RESIZE"` in `wezterm.lua` |

## Manual binding (lazy-spawn)

```ini
bind = SUPER, Z, exec, ~/.config/dino/dino-toggle.sh

windowrule = workspace special:dino silent, class:^(dino)$
windowrule = float, class:^(dino)$
windowrule = size 60% 50%, class:^(dino)$
windowrule = center 1, class:^(dino)$
```

The toggle script (generated by `dino setup`) checks if dino is already running and either launches it or just toggles the workspace.

## Choosing a different hotkey

`SUPER+Z` was picked to avoid the most common conflicts. Free single-key candidates on a typical Hyprland setup are `Z`, `Y`, `U`, `H`. With modifiers, `SUPER+SHIFT+V` and `SUPER+SHIFT+D` are usually free even on heavy configs.

To check what's free on your machine:

```bash
grep -E '^[[:space:]]*bind[d]?[[:space:]]*=[[:space:]]*[^,]*Super' ~/.config/hypr/**/*.conf
```

## Environment variables under Hyprland

If you launch Hyprland from SDDM/gdm/login manager, your shell rc is **not** sourced. The cleanest fix is the config file (`dino setup` already does this). Alternative: export inside Hyprland:

```ini
env = OPENAI_API_KEY,sk-...
```

## Troubleshooting

### Nothing happens when I press `SUPER+Z`

Make sure dino is running. With exec-once it spawns at login; you can check:

```bash
hyprctl clients | grep -A2 'class: dino'
```

If nothing prints, either (a) the exec-once didn't run (typo in the command, terminal missing) or (b) the window class regex doesn't match. Verify with:

```bash
hyprctl monitors      # confirm the special workspace mechanism works on your hyprland version
```

### `wtype` typed nothing into Electron apps

That was a v0.1 problem. In v0.2 the default output is `wl-copy` + manual `Ctrl+V`, which works universally. If you still want `wtype`, set `[output].adapter = "wtype"` in `~/.config/dino/config.toml`.

### "No se encontró micrófono" inside the TUI

Verify PipeWire is capturing from the right source:

```bash
wpctl status               # confirm there's a default audio source
pw-record /tmp/test.wav    # record 3 s, Ctrl+C, then `aplay /tmp/test.wav`
```

If the wrong source is captured, set it as default in `wpctl` or `pavucontrol`.

### `wl-copy` clipboard disappears when I close dino or log out

`wl-copy` forks by default — the clipboard survives until something else replaces it or the Wayland session ends. To keep it alive across logouts and reboots, run [`wl-clip-persist`](https://github.com/Linus789/wl-clip-persist) as a background daemon.

`dino setup` checks for `wl-clip-persist`: if it's installed it offers to add `exec-once = wl-clip-persist` to your `hyprland.conf`; if it isn't, the wizard prints the right install command for your distro.

**Manual install (Arch, AUR):**

```bash
paru -S wl-clip-persist-bin    # or: yay -S wl-clip-persist-bin
```

**Manual install (Debian/Ubuntu/Fedora/openSUSE — no official package):**

```bash
cargo install wl-clip-persist
# or grab a precompiled release from
# https://github.com/Linus789/wl-clip-persist/releases
```

**Wire it into Hyprland yourself:**

```ini
exec-once = wl-clip-persist
```

## Legacy CLI (v0.1 style)

The fire-and-forget push-to-talk commands still work and now also output via `wl-copy`:

```ini
bind  = SUPER, V, exec, dino start
bindr = SUPER, V, exec, dino stop
```

Use this if you don't want a persistent TUI window. It's not the recommended path, but it stays supported through v0.3 and will be re-evaluated for v1.0.
